# 统一实施计划：五分钟暴兵、波次构筑与金币商店

## 1. 目标与当前基线

本计划取代此前的经验曲线方案，并继续承接已经完成的战斗反馈、移动相机、升级文字与工业硬核 BGM。最终体验是一个约五分钟的高密度割草闭环：传送门在玩家周围安全环带内集中暴兵，清空一波后进入统一结算，玩家先免费领取一次流派奖励，再用本波收集的金币购买补强；手动暂停只负责继续游戏。连杀超载提供短时无敌和超高输出。

已经完成并保留：

- 单一伤害/击杀事实、受限 hit-stop、击杀确认、镜头反馈。
- 稳定移动与相机跟随、冲刺基础、升级选项中文可读性。
- 168 BPM 工业硬核 BGM 与基础音量控制。
- 自动化测试入口：`powershell -ExecutionPolicy Bypass -File scripts/tests/run_tests.ps1`。

## 2. 核心循环

```text
传送门预警 -> 多门暴兵 -> 高速清场 -> 金币散落与吸附拾取
      -> 清剿完成提示 -> 三流派各两卡统一结算
      -> 免费领取一张 -> 可继续付费购买 -> 关闭并进入下一波
      -> 连杀充能 -> 短时无敌超高输出 -> 最终首领/结算
```

成长通过同一结算界面提供两个互补通道：

- 波次升级：免费、每个可继续作战的已清波次一次，主要决定弹幕、无人机/电弧、冲刺/地刺三条流派及终极进化。
- 金币购买：领取免费奖励后消费本局金币，主要提供可控补强、缺口修复和流派协同；结算快照稳定，不能刷新报价。

终局波清空后直接结算，不弹出没有后续战斗价值的升级选择。

## 3. 产品验收指标

- 单局中位时长目标 `5:00`，正常范围 `4:30–5:30`。
- 玩家在前两次波次升级内看到明确流派方向，终局前获得一次终极进化。
- 每个非终局波次清空后恰好获得一次免费三选一；重复事件不能重复奖励。
- 敌人不再提供经验；死亡只生成一次金币掉落和既有护盾掉落。
- 金币拾取只结算一次，本局余额不能为负；统一结算重复打开不能生成新报价或免费刷新。
- 每次攻势在玩家周围安全环带生成多个传送门，主要敌群在短时间内集中涌出。
- 超载持续时间短，完整窗口无敌且高输出，音画起止清晰，退出后临时倍率完全恢复。
- 阶段快照包含金币、升级、商店报价状态和待开始阶段；不保存屏幕中的敌人、弹幕、掉落与 VFX。

## 4. 架构决策

### 4.1 单一击杀收益事实

`Enemy` 只携带 `coin_value`；`WaveDirector` 只发出一次 `enemy_killed(enemy, source, coin_value)`，并延迟生成一次金币掉落。`Main` 负责把拾取结果交给 `UpgradeSystem`，表现层不得自行增加余额。

### 4.2 成长状态归属

第一版继续由 `UpgradeSystem` 持有本局成长状态，避免在核心链路未稳定前拆出空泛框架：

```gdscript
signal progression_state_changed(state: Dictionary)
func add_coins(amount: int) -> bool
func spend_coins(amount: int) -> bool
func prepare_settlement(completed_wave: int) -> bool
func claim_free_offer(request: Dictionary) -> bool
func purchase_settlement_offer(request: Dictionary) -> bool
func complete_settlement(request: Dictionary) -> bool
```

`family_levels` 分别表示火力、机动、工程等级，初始均为一；只有免费奖励提升对应流派等级。每次结算使用不可伪造、不可重复消费的事务号，付费购买只增加卡牌层数。

### 4.3 波次推进所有权

`WaveDirector` 在清空波次后进入 `waiting_for_advance`，只发一次 `wave_finished`。`Main` 依次控制清剿提示和统一结算；合法关闭结算后显式调用 `advance_after_settlement()`，再由下一次 `wave_prepared` 打开开场提示。这样下一波不会在提示或选择界面背后提前生成。

### 4.4 商店报价稳定性

统一结算为每个流派生成两张稳定报价，重复打开不刷新，已领取/购买项标记售罄。报价来自同一卡牌目录，并携带 `family`、`kind`、`cost`、`max_rank` 与事务号。领取免费奖励后才允许继续付费购买或关闭；手动暂停不再提供报价入口。

### 4.5 存档边界

只在稳定边界保存：免费奖励完成后、结算购买完成后、准备进入下一阶段时。快照保存可重建数据，不保存运行时节点。损坏或未知版本安全回退到新游戏。

## 5. 分阶段任务

### Phase 1：波次升级与金币基础闭环

#### Task 1.1：替换经验成长契约

验收：

- `UpgradeSystem` 不再暴露经验阈值；金币加减安全，余额不为负。
- 每个有效非终局波次只能排队一次免费升级；应用升级后等级增加一次。
- 伪造、重复或封顶选项继续被拒绝。

验证：`UpgradeTest`、`BalanceTest`。

#### Task 1.2：建立可暂停的波次结算门

验收：

- 波次清空只发一次 `wave_cleared`，等待升级期间不启动下一波。
- 升级完成后显式推进；最终波直接胜利且不留下待选升级。
- 现有生成帧率独立性、敌人注册表与胜利路径不回归。

验证：`WaveTest`、`StateTest`、`SmokeTest`。

#### Task 1.3：金币掉落、拾取与 HUD

验收：

- 所有 `xp_value`/`ExperienceShard` 运行时路径替换为金币语义。
- 单次击杀只生成一枚对应价值的金币掉落，单枚掉落只结算一次。
- HUD 显示“流派等级”和“金币”，不再显示经验进度条。

验证：`CombatEventTest`、`ProjectilePickupTest`、`UITest`、`PerformanceTest`。

### Checkpoint A：第一阶段

- 完整测试通过，无 XP 运行时引用。
- 手动确认金币可见、可吸附、余额变化；清空波次后升级界面出现且选择后进入下一波。
- 第一阶段形成聚焦提交并推送。

### Phase 2：暂停商店闭环（历史基线，Phase 3 已统一替换）

#### Task 2.1：共享升级目录与商店报价契约

- 为升级补齐流派、核心/支援类型、费用、前置和封顶信息。
- 每波稳定生成三项报价；暂停切换不重抽，购买后售罄。
- 费用随阶段温和增长，余额不足或非法事务不改变状态。

#### Task 2.2：暂停商店 UI

- 暂停界面同时提供继续按钮、余额和三项可购买升级。
- 明确显示费用、流派、说明、售罄/余额不足状态，键盘与鼠标均可操作。
- 购买后立即更新玩家属性、余额、报价和反馈。

#### Task 2.3：商店状态集成

- 暂停所有权仍只由 `Main` 状态机管理。
- 购买不会关闭暂停界面；恢复后临时战斗状态保持干净。
- 商店购买与免费波次升级共享效果实现，但使用独立事务和统计。

### Checkpoint B：双通道成长历史基线

- 免费波次升级与金币购买都能稳定改变构筑，互不重复结算。
- 反复暂停不能刷报价或增加金币。
- 960×540 至 2560×1080 均可读可操作。

### Phase 3：统一结算、三流派构筑与战斗重量

详细合同与数值表见 `docs/specs/2026-07-19-phase3-settlement-builds-combat.md`。

#### Task 3.1：战斗反馈重量分级

- 轻型击杀使用小震屏；中型保持当前力度；大型使用完整震屏。
- 枪声替换电子提示音；轻/中/大型受击音按重量区分。
- Bruiser 基础生命调整到 `300`，但接触伤害不同时提高。

#### Task 3.2：远程敌人可见交战区

- Spitter 出生在相机安全区与玩家距离环的交集内。
- 离开可见安全区时全速返回；靠边且玩家过近时沿边横移，不向屏幕外逃。
- 未进入可见交战区前不能开火。

#### Task 3.3：波次提示与显式推进

- 状态机按 `WAVE_INTRO -> PLAYING -> WAVE_CLEAR -> SETTLEMENT` 推进。
- 开始提示结束前不生成敌人；完成提示结束前不显示结算。
- 最终波也显示完成提示，但之后直接进入胜利结算。

#### Task 3.4：三流派六卡统一结算

- 火力、机动、工程每个流派稳定生成两张卡。
- 先免费领取一张，之后才能付费购买和关闭；三个动作使用独立防伪事务。
- 手动暂停不再提供第二条购买入口；点击关闭才允许下一波开始。

#### Task 3.5：流派等级与卡池重平衡

- 三流派等级初始均为 `1`；免费奖励提升对应等级，付费只升卡牌层数。
- 普通卡单层收益控制在 `8%–20%`，射速、移速、地刺寿命、无人机数等均有硬上限。
- 地刺与移速/冲刺、无人机与电弧、枪线与弹速/穿透形成可读联动。

#### Task 3.6：三条终极进化

- 火力：轨道风暴。
- 工程：雷网矩阵。
- 机动：裂地超载。
- 第四波后且对应流派等级达到 `5` 才进入候选池；每局最多一个终极进化。

### Checkpoint C：构筑深度

- 四种目标分辨率下完整通过“提示、战斗、提示、免费、购买、关闭、下一波”。
- 三条路线在相同免费投资下输出量级接近，并在前两次结算形成明确方向。
- 每波通常可多买一张，存钱可买两张，不能稳定买空剩余五张。
- 轻重敌人反馈、Bruiser TTK 与 Spitter 可见性达到规格中的自动化边界。

### Phase 4：传送门暴兵与连杀超载

#### Task 4.1：传送门组件与安全环采样

- 传送门状态为预警、暴兵、关闭；硬安全距离不低于 `260px`。
- 边界回退仍满足安全距离，单帧生成受预算限制。

已完成：程序化霓虹传送门组件提供“预警 → 暴兵 → 关闭”生命周期；`WaveDirector` 安全环采样保证世界边界与 `260px` 最小距离。敌人从门内分批涌出将在 Task 4.2 接入。

#### Task 4.2：多门集中攻势

- 每次攻势生成多个门并在短窗口集中涌敌，不再全地图零散刷新。
- 活跃敌人、队列和门均清空后才允许波次结算。

#### Task 4.3：连杀超载状态

- 连杀充能、短时触发、全程无敌、全伤害高倍率和高射速倍率有单一状态源。
- 退出、暂停、升级、结算与重启后所有临时倍率归零。

#### Task 4.4：超载音画与 HUD

- 触发重击、短循环、结束尾音与角色高亮、边缘能量、受限震屏明确区分起止。
- 高密度输出下不产生无限音频播放器、VFX 节点或常驻 hit-stop。

### Checkpoint D：爽感核心

- 暴兵具有集中压迫感但保留公平预警。
- 超载短、稀有、压倒性，且玩家能明确理解无敌和高输出窗口。

### Phase 5：五阶段、首领与继续游戏

#### Task 5.1：五阶段节奏、远程敌人扩展与最终首领

- 用五个阶段替换旧八波数量表；最终首领具有预警攻击和传送门援军。
- 在 Phase 4 传送门和生成预算稳定后加入 Marksman 与投射型远程敌人；保留 Spitter 作为中距离基础远程单位。
- 标准构筑中位时长进入 `4:30–5:30`。

#### Task 5.2：版本化阶段快照

- 保存金币、等级、升级计数、进化、稳定商店报价、玩家基础状态、击杀和用时。
- 原子写入，拒绝损坏、非法数值和未知版本。

#### Task 5.3：继续游戏流程

- 有效快照提供继续入口，从待开始阶段重建。
- 胜利、失败和主动新游戏清理旧档；中途退出保留最近稳定快照。

### Checkpoint E：五分钟闭环

- 新游戏、暂停购买、波次升级、保存、继续、胜负与重开形成完整闭环。
- 不恢复敌人、弹幕、门、掉落、VFX、音频循环或相机偏移。

### Phase 6：平衡、性能与最终验收

#### Task 6.1：经济与构筑平衡

- 记录各阶段金币收入、购买率、首次成型时间、进化达成率和三条路线通关率。
- 商店价格让玩家每阶段通常能做一项有意义取舍，而不是买空或完全买不起。

#### Task 6.2：密集战斗性能

- 250 敌人、最大传送门攻势与最高弹幕不产生无上限节点、RID 或音频播放器。
- 30/60/120 Hz 下关键计时与生成数量保持容差。

#### Task 6.3：可访问性与发布回归

- HUD、统一结算、纯暂停页和结果页在四种目标分辨率下可读。
- 震屏/闪光强度只影响表现，不影响规则；Windows 导出启动通过。

## 6. 依赖与并行边界

```text
Phase 1 成长/波次/金币契约
  -> Phase 2 暂停商店
  -> Phase 3 流派与进化
  -> Phase 5 存档数据格式

Phase 1 波次结算门
  -> Phase 4 传送门攻势
  -> Phase 5 五阶段/首领

既有击杀事实 + 有效属性/免疫合同
  -> Phase 4 连杀超载
```

Phase 2 的商店 UI 可在报价合同锁定后与目录元数据并行；Phase 4 的传送门线和超载线可在各自不同时修改 `Main.gd` 的前提下并行。涉及 `Main.gd`、`WaveDirector.gd`、`UpgradeSystem.gd`、`Player.gd` 的集成任务必须按依赖顺序合并。

## 7. 风险与控制

| 风险 | 影响 | 控制措施 |
|---|---|---|
| 波次清空重复发奖 | 高 | `waiting_for_advance` + 已奖励波次去重 + 重复事件测试 |
| 旧 XP 与金币双重结算 | 高 | 删除 XP 运行时路径；单一击杀收益事实；全仓检索门禁 |
| 暂停刷商店报价 | 高 | 报价按波次持久化；暂停/恢复只展示，不重新生成 |
| 金币过多导致商店无取舍 | 中 | 记录阶段收入分位数；成本随阶段增长；购买率回归测试 |
| 商店稀释流派方向 | 中 | 免费升级偏核心，商店偏支援；家族权重与里程碑保底 |
| 升级暂停背后提前开波 | 高 | `WaveDirector` 显式等待，只有完成升级才推进 |
| 超载临时倍率污染永久属性 | 高 | 基础值/有效值分离；状态退出和重启回归 |
| 暴兵同帧卡顿 | 高 | 门级与全局生成预算、活跃敌人上限、性能基线 |

## 8. 完成标准

- 所有阶段验收标准与检查点通过，任务清单状态同步。
- 每项新行为先有失败测试，再完成实现；完整测试无错误、泄漏或跳过。
- 每个纵向切片形成聚焦提交并推送，不包含凭据、生成状态或无关改动。
- 三条构筑、金币商店、传送门暴兵、超载、保存继续和五分钟时长完成手动回归。
