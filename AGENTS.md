# Project Workflow

After completing and verifying a change, create a focused Git commit and push the current branch to `origin` automatically.

Before each push:

- Confirm the commit contains only completed, relevant work.
- Run the most relevant available verification and report failures honestly.
- Do not push secrets, credentials, generated local state, or unrelated in-progress edits.
- Do not change repository visibility, remotes, branch protection, or rewrite history without explicit user approval.
