# Phase 3 规格：波次结算、三流派构筑与战斗重量

## 目标

把当前分离的免费三选一与暂停商店重构为唯一的波次结算闭环，同时修正轻重敌人的震屏、受击音、枪声音效和远程敌人可见性。

目标体验：

```text
侦测到第 X 波敌人
  -> 提示结束后开始出敌
  -> 第 X 波清剿完成
  -> 三流派各两张卡的统一结算
  -> 免费领取一张
  -> 可用金币购买其余卡
  -> 点击关闭
  -> 下一波提示
```

## 已确认假设

- 最新的波次结算需求取代战斗中暂停购买；手动暂停只负责继续游戏。
- 第一版保留三条流派：`ballistics`（火力）、`mobility`（机动）、`automation`（工程）。
- 三条流派等级初始均为 `1`。只有每波免费奖励提升对应流派等级；金币购买只提升卡牌层数。
- 每级为对应伤害源提供 `+5%` 乘算加成：火力对应子弹，机动对应冲刺/地刺，工程对应激光/电弧。
- 最终波显示“清剿完成”后进入胜利结算，不生成没有后续战斗价值的卡牌结算。
- 新增远程敌人种类延后到 Phase 5；当前 Phase 3 只修复 Spitter 的出生、可见距离和移动逻辑。

## 技术与命令

- 引擎：Godot 4.7，GDScript。
- 完整验证：`powershell -ExecutionPolicy Bypass -File scripts/tests/run_tests.ps1`
- 源码：`scripts/`；场景：`scenes/`；测试：`scripts/tests/`；规格：`docs/specs/`。

## 状态机

`Main` 是唯一运行状态所有者：

```text
START -> WAVE_INTRO -> PLAYING -> WAVE_CLEAR -> SETTLEMENT -> WAVE_INTRO
                       |            |
                       |            +-> RESULT（最终波）
                       +-> PAUSED -> PLAYING
```

- `WAVE_INTRO`：世界暂停，显示“侦测到第 X 波敌人”；提示完成后显式开始波次。
- `PLAYING`：生成与战斗运行；只有此状态能打开手动暂停。
- `WAVE_CLEAR`：世界暂停，显示“第 X 波清剿完成”。
- `SETTLEMENT`：世界暂停，统一处理免费领取、金币购买和关闭。
- `PAUSED`：不提供购买入口。
- 下一波只能由一次成功的结算关闭事务推进。

`WaveDirector` 不再通过隐含倒计时自行开始下一波：

```gdscript
signal wave_prepared(summary: Dictionary)
signal wave_finished(summary: Dictionary)

func prepare_next_wave() -> bool
func begin_prepared_wave() -> bool
```

`summary` 至少包含 `wave`、`total`、`is_final`。最终波也必须产生一次 `wave_finished`。

## 统一结算合同

`UpgradeSystem` 是卡牌目录、流派等级、金币与交易的单一事实源：

```gdscript
signal progression_state_changed(state: Dictionary)
signal settlement_changed(state: Dictionary)

func prepare_settlement(completed_wave: int) -> bool
func get_settlement_state() -> Dictionary
func claim_free_offer(request: Dictionary) -> bool
func purchase_settlement_offer(request: Dictionary) -> bool
func complete_settlement(request: Dictionary) -> bool
```

稳定状态形状：

```gdscript
{
    "wave": 3,
    "coins": 82,
    "transaction": 3,
    "reward_claimed": false,
    "can_close": false,
    "family_levels": {"ballistics": 2, "mobility": 1, "automation": 1},
    "families": [
        {"id": "ballistics", "label": "火力", "offers": [offer_a, offer_b]},
        {"id": "mobility", "label": "机动", "offers": [offer_c, offer_d]},
        {"id": "automation", "label": "工程", "offers": [offer_e, offer_f]},
    ],
}
```

交易规则：

- 每个流派恰好生成两张不同、合法、未封顶的卡，共六张。
- 同一结算重复打开时，卡片、价格、顺序和事务号完全不变。
- 免费奖励未领取前拒绝付费和关闭；第一次合法点击执行免费领取。
- 免费领取和付费购买是两个独立接口，不能通过调用者传回的价格或 `cost = 0` 模拟。
- 请求只信任内部匹配的卡牌 ID 与事务号；重复、伪造、过期请求无副作用。
- 免费领取后允许购买其余卡；关闭只成功一次。

## 流派与卡牌平衡

普通卡单层边际收益目标为 `8%–20%`；机制解锁卡价格更高、层数更低。所有乘算成长都有硬上限。

| 卡牌 | 单层效果 | 上限 | 基础价 |
|---|---|---:|---:|
| 超频弹芯 | 子弹伤害 `+12%` | 5 | 32 |
| 灼热枪管 | 射速 `+10%` | 5 | 34 |
| 线圈加速器 | 弹速 `+20%`，子弹伤害 `+5%` | 4 | 28 |
| 轨道穿甲 | 穿透 `+1`，子弹伤害 `+4%` | 4 | 40 |
| 分裂枪膛 | 枪线 `+1`，单发伤害 `-8%` | 2 | 62 |
| 攻城弹头 | 子弹伤害 `+20%`，射速 `-5%` | 3 | 46 |

### 机动 `mobility`

| 卡牌 | 单层效果 | 上限 | 基础价 |
|---|---|---:|---:|
| 伺服腿甲 | 移速 `+8%`，地刺伤害 `+6%` | 5 | 30 |
| 静滞地刺 | 首次解锁；后续伤害 `+15%`、持续 `+0.5s` | 5 | 52 |
| 裂地密度 | 地刺间距 `-12%` | 4 | 38 |
| 冲刺冷却 | 冲刺冷却 `-12%`，冲刺伤害 `+8%` | 4 | 38 |

移速与地刺形成天然联动：地刺按移动距离生成，移动越快，单位时间铺设的地刺越多；冲刺距离进一步形成瞬时地刺走廊。

### 终极进化

- 火力：`轨道风暴`，高流派等级后周期性发射环形副弹幕。
- 工程：`雷网矩阵`，强化无人机/电弧覆盖并形成高频电网。
- 机动：`裂地超载`，显著强化冲刺频率、距离与地刺走廊。
- 第 4 波后、对应流派等级达到 `5` 才进入该流派候选池；每局最多获得一个终极进化。

## 经济目标

- 敌人金币不再随波次额外乘算；数量增长本身提供更多收入。
- 卡牌价格随完成波次按 `1 + 0.55 × (wave - 1)` 增长。
- 目标是每波通常可额外购买一张；存钱后可买两张；不能稳定买空剩余五张。

## 战斗反馈与敌人

- 轻型（Dasher/Spitter）击杀震屏 `0.28`；中型（Scrapper）保持 `0.80`；大型（Bruiser）`1.00`。
- Bruiser 基础生命从 `250` 调到 `300`；第二波实际生命为 `378`，基础枪械击杀时间目标 `2.7–3.1s`。
- 轻型受击音为短、高频、轻噪声；大型受击音为 `0.12–0.14s` 的低频复合冲击。
- 发射声改为约 `0.07s` 的程序化枪声，包含起始爆音、低频枪身和快速衰减的中频裂响。
- Spitter 必须生成在玩家相机安全可见区，保持动态远距离环；离开安全区时全速返回，未进入可见交战区前不射击。

## UI 与可访问性

- 新建独立 `SettlementScreen`，三列流派、每列两张卡；不继续扩大暂停页。
- 免费领取前，卡片显示“免费奖励”，关闭按钮禁用；领取后剩余卡显示价格和可购买状态。
- HUD 格式：`等级  火力:x  机动:y  工程:z`。
- 波次大字位于屏幕上方，中文字体、透明度和缩放动画不依赖世界暂停。
- 鼠标、数字键 `1–6`、键盘/手柄焦点均可完成结算。
- 目标分辨率：`960×540`、`1280×720`、`1920×1080`、`2560×1080`。

## 测试策略

- `CombatFeedbackTest`：轻/中/大型震屏分级、击杀反馈保留。
- `BalanceTest`：枪声包络、轻重受击 WAV、Bruiser TTK、卡牌边际收益和硬上限。
- `EnemyBehaviorTest`：三种目标分辨率的可见安全区、返回速度、边缘逃逸与开火门。
- `SettlementTest`：六卡稳定快照、免费/付费/关闭三个事务、伪造与重复请求。
- `WaveTest` / `StateTest`：开始提示前不生成、完成提示先于结算、关闭前不推进、最终波路径。
- `UITest`：六卡中文、布局、焦点和数字键。
- `SmokeTest`：完整“开始提示 → 战斗 → 完成提示 → 免费 → 购买 → 关闭 → 下一波”。

## 边界

- 始终：先写失败测试；每个纵向切片完整回归后独立提交并推送。
- 需要用户授权：新增第三方依赖、改变远端或历史、扩大到发布/存档格式迁移。
- 禁止：信任 UI 价格、通过重复事件发奖励、恢复两条购买入口、提交生成状态或凭据。

## 本轮不做

- Marksman 与投射型远程敌人：加入 Phase 5，等待 Phase 4 传送门和生成预算稳定。
- 五阶段波表、存档继续与最终首领：仍按原 Phase 5 顺序实施。
- 传送门暴兵与连杀超载：仍在 Phase 4；本轮只保证新结算合同不阻塞它们。

## 验收

- 六项用户修改除“新增远程敌人种类”按约定进入后续计划外均有自动化覆盖并可试玩。
- 三条流派在相同免费投资下输出量级接近，前两次结算能看出明确方向。
- 完整测试无错误、泄漏或跳过；提交只包含相关工作并推送到当前分支。
